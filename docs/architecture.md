# 架构说明

## 总体架构

系统采用前后端分离架构：

- 前端通过 Vue 3 + Element Plus 提供铁路政企风格后台界面。
- 后端通过 Spring Boot 3 提供 RESTful API。
- MySQL 保存用户、组织、权限、文件元数据、日志和通知。
- Redis 保存验证码和后续可扩展的会话状态。
- MinIO 保存真实文件内容。
- Nginx 负责前端静态资源和 API 反向代理。

## 权限模型

权限分为两层：

- 系统级权限：控制功能入口，例如文件上传、文件删除、用户管理、组织管理、日志查看。
- 文件访问权：控制用户能不能访问某个具体文件。

第一阶段不拆分文件级别的查看、下载、预览、编辑、抄送权限。用户只要拥有某文件访问权，就默认可以查看、预览、下载、在线编辑和抄送。

## 文件访问权来源

- 超级管理员拥有全部文件访问权。
- 文件上传人默认拥有访问权。
- 文件可见范围 `visibility_scope`。
- 文件授权表 `sys_file_permission`。
- 文件抄送表 `sys_file_copy`。

统一通过 `FileAccessService.canAccess(userId, fileId)` 判断。

## 存储设计

MySQL 只保存文件元数据，不保存文件二进制内容。真实文件默认存储到 MinIO。后续可通过 `FileStorageService` 扩展本地存储、NAS 或其他对象存储。

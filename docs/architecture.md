# 架构说明

## 总体架构

系统采用前后端分离架构：

- 前端通过 Vue 3 + Element Plus 提供铁路政企风格后台界面。
- 后端通过 Spring Boot 3 提供 RESTful API。
- MySQL 保存用户、组织、权限、科室资料目录、附件元数据、日志和通知。
- Redis 保存验证码和后续可扩展的会话状态。
- MinIO 保存真实文件内容。
- Nginx 负责前端静态资源和 API 反向代理。

## 权限模型

权限分为两类：

- 系统级权限：控制功能入口，例如用户管理、组织管理、角色管理、日志查看、存储查看。
- 资料业务权限：科室用户维护本科室多级资料目录和文件入口，科室最高级文件夹仅本科室管理员和超级管理员可新建，车间用户查看全部科室资料并上传本车间附件，超级管理员拥有全部权限。

## 资料业务模型

- `sys_doc_node` 表示科室下的多级资料目录节点，节点类型包含文件夹和文件。
- `sys_doc_item` 表示文件入口，包含文件年份，并按 `business_type` 分为上传任务和下达文件，文件节点通过 `item_id` 关联文件入口。
- 上传任务通过 `sys_doc_upload_requirement` 配置一个或多个收集项，用户完整提交后写入 `sys_doc_submission`，附件保存到 `sys_doc_attachment` 并记录对应收集项。
- 下达文件通过富文本内容和 `sys_doc_item_attachment` 承载公示附件，不参与上传任务进度统计。

## 存储设计

MySQL 只保存附件元数据，不保存附件二进制内容。真实附件默认存储到 MinIO。后续可通过 `FileStorageService` 扩展本地存储、NAS 或其他对象存储。

# 上线前傻瓜式手工测试方案

本文档用于上线前人工验收。目标是从“只有默认超级管理员账号”开始，按真实业务路径完成账号注册、审批、管理员配置、权限配置、资料创建、车间上传和多用户权限隔离验证。

## 1. 测试目标

- 验证默认超级管理员能完成全局初始化、审批、设管理员和权限配置。
- 验证科室管理员能维护本科室资料、审批本科室用户、查看本科室上传记录。
- 验证车间用户能按任务上传资料，但不能查看他人提交。
- 验证跨科室、跨车间、无权限页面直达访问都被拦截。
- 验证文件预览、下载、回收站、日志、存储统计等上线前主功能。

## 2. 管理员配置原则

建议配置 3 个管理员：

| 管理员 | 必测目的 | 少配后的风险 |
|---|---|---|
| 技术科管理员 | 测试科室资料维护、上传任务创建、提交记录查看、大修模板库维护 | 无法验证核心资料维护链路 |
| 安全科管理员 | 测试跨科室不能维护技术科资料 | 容易漏掉跨科室越权问题 |
| 房建车间管理员 | 测试车间侧账号审批和用户管理范围 | 容易漏掉车间管理员只能管理本车间用户的问题 |

时间紧时最低保留 2 个管理员：技术科管理员、安全科管理员。房建车间普通用户可由默认超级管理员审批，但会少测“车间管理员审批本车间用户”。

## 12. 通过标准

### 12.1 阻断上线

出现以下任一问题，建议阻断上线：

- 登录、注册、审批主链路失败。
- 默认超级管理员无法设管理员。
- 科室管理员无法维护本科室资料。
- 车间用户无法提交上传任务。
- 文件上传或下载失败。
- 跨科室、跨车间、他人提交记录存在明显越权。
- 普通用户可访问管理页或敏感日志。

### 12.2 必须修复或明确风险

- 文件预览异常。
- 回收站恢复异常。
- 日志缺关键操作记录。
- 存储统计不准确。
- 权限变更后不重新登录也长期不生效，且页面无提示。
- 用户错误提示暴露端口、SQL、Java 异常、MinIO 等底层信息。

### 12.3 可评估带缺陷上线

- 样式间距、按钮位置、表格列宽等非阻断问题。
- 搜索体验不够顺滑但功能可用。
- 统计刷新有短暂延迟但刷新页面后正确。
- 个别提示文案不够准确但不影响操作。

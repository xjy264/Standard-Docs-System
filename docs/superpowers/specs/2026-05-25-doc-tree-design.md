# 多级资料目录树设计

## 目标

将科室资料展示页从“二级菜单 + 文件表格”改为单一科室侧边栏加右侧多级资料目录树。目录最高五层，每一级都可以放文件夹和文件；普通用户只读，管理员和超级管理员可以维护目录和文件。

## 数据模型

新增 `sys_doc_node` 作为统一目录节点表，节点类型分为 `FOLDER` 和 `FILE`。文件节点通过 `item_id` 关联 `sys_doc_item`，文件详情、富文本内容、附件上传开关和上传记录继续复用现有文件入口模型。

为解除新文件对旧二级菜单的强依赖，`sys_doc_item` 新增 `section_dept_id`，`category_id` 允许为空。旧数据按原 `sys_doc_category.section_dept_id` 回填 `section_dept_id`，旧二级菜单迁移为一级文件夹，旧文件入口迁移为其下的文件节点。

## 接口

- `GET /api/doc-tree?sectionDeptId=1`：返回科室下的多级目录树。
- `POST /api/doc-nodes/folders`：新增文件夹。
- `POST /api/doc-nodes/files`：新增文件，同时创建 `sys_doc_item` 和文件节点。
- `PUT /api/doc-nodes/{id}`：修改节点名称、排序和父级；文件节点同步更新文件入口名称。
- `DELETE /api/doc-nodes/{id}`：删除节点；文件夹存在子节点时禁止删除。

## 前端交互

`OrgFilesView.vue` 保留左侧一级科室导航，右侧改为折叠式目录树。文件夹显示加号展开；文件显示为可点击标题，进入现有文件详情页。管理员和超级管理员显示新增、编辑、删除操作；普通用户不显示维护入口。

## 权限与约束

维护权限沿用现有规则：超级管理员可维护全部科室，本科室管理员可维护本科室资料，普通用户不可新增、修改或删除。目录层级超过五层时后端拒绝保存。文件夹存在子节点时禁止删除，避免误删整棵目录树。

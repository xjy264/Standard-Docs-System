# 需求说明

## 项目目标

建设“大同房建公寓段标准化资料管理系统”，用于统一配置内业资料和规章制度两类资料目录，维护文件内容，展示制度文件，并收集车间在内业资料目录下上传的标准化资料附件。

第一阶段按真实 MVP 实施，重点保证半个月内可演示注册审批、组织管理、用户权限矩阵、首页模块入口、科室多级目录维护、文件发布、车间在内业资料目录上传文件、历史上传记录查看和操作日志。

## 一期范围

- 登录、注册、滑块验证码、管理员审批。
- 登录态使用 HttpOnly Cookie 保存，前端不保存登录 JWT；系统不提供跳过密码的管理员免密登录入口。
- 组织树、部门管理、用户管理、角色模板。
- 用户系统级权限配置和权限矩阵。
- 首页不展示侧边栏，只保留“内业资料”和“技术规章、文件、管理办法”两个蓝色大字入口，不展示说明小字；顶部导航左侧提供“首页”按钮返回首页。
- 内业资料侧边栏固定为九个科室；规章制度模块先展示“技术规章、技术文件、管理办法”，再按同样顺序展示九个科室。固定侧边栏不允许在组织管理中改名、禁用或删除；三个规章专用入口不作为用户所属部门。
- 内业资料和规章制度使用各自独立目录树，历史目录和文件默认归属内业资料，规章制度从空模块开始由科室维护。
- 科室多级资料目录维护，最高五层，每一级可放文件夹和文件。
- 目录树下文件入口维护、文件年份由年度目录继承，并维护富文本文件内容；编辑文件时可拖拽或点击重新选择文件附件。
- 科室资料页默认按当前年份展示文件列表，并支持按年份和文件名搜索。
- 科室资料页使用统一目录展示上传任务和下达文件，不再展示“上传 / 下达”tabs。
- 内业资料文件夹可开启“使用完成进度条”并填写大于零的完成目标数；进度分子只统计该文件夹的直接、未删除且当前用户可见文件，子文件夹内文件不计入。文件数可超过目标数，条形宽度封顶 100%，文字仍显示实际的“当前数/目标数”。规章制度模块不提供该进度配置。
- 文件详情支持预览 PDF、图片、Word、Excel 和 PPT；其中 PDF 和图片直接内联预览，Office 文件通过 OnlyOffice 预览服务展示。
- Office 预览使用短时附件访问票据，票据只允许读取对应附件，不暴露登录 JWT。
- 科室维护的文件正文附件同一时间只允许保留一个；如需重新上传，必须先删除原有正文附件。
- 车间用户可在内业资料任意文件夹下新增文件，新文件直接保存在当前父文件夹，并记录所属车间；不再通过“允许车间上传”或“可上传车间”限制新增，也不再自动生成车间名称文件夹。
- 历史自动生成的车间文件夹和历史提交记录保留兼容展示，不做数据删除或迁移。
- 规章制度模块只用于展示制度文件，科室和超级管理员维护文件夹及文件，车间只查看、预览和下载，不提供车间上传配置。
- 历史“文件=上传清单”的上传任务数据保留在内业资料模块，只读查看和下载历史提交记录；默认不再暴露新的文件级上传清单入口。
- 单份提交任务按车间锁定，每个选中车间最多保留一条当前有效提交；同车间普通用户可查看是否已上传但不能重复上传，科室用户、超级管理员、本车间管理员和上传本人可下载，车间侧不能删除，科室用户和超级管理员可删除用于纠错。
- 多份提交任务允许每个用户上传多条提交；上传本人可删除自己的提交，本车间管理员可删除本车间提交，科室用户和超级管理员可管理任务下全部提交。
- 科室用户和超级管理员查看本科室上传任务下全部上传记录；普通上传者只能查看和下载自己有权限的提交附件。
- 下达文件用于科室公示信息，支持富文本内容和下达附件下载，不参与上传进度统计。
- 科室文件删除后进入本科室共享回收站，回收站按文件表格展示，不做目录分级；恢复时由用户选择同科室目标目录。回收站文件保留 30 天，超过期限后系统自动清理正文附件、车间上传附件和相关元数据；单独软删除的正文附件超过 30 天后也自动清理。
- 首页展示两个模块入口，不再展示科室组织树统计卡片。
- 首页统计、操作日志、登录日志、存储统计基础版。
- 通知台提供超级管理员专属“系统错误”页签，统计后端异常、前端运行错误和接口错误；支持按错误编号追踪、标记处理和一键导出故障包，便于内网部署现场排查。
- 用户侧错误提示面向普通用户，不暴露 HTTP 状态码、程序异常、端口号或底层服务名称；同类并发错误只提示一次。
- Docker Compose 部署。

## 延后范围

- 大文件分片上传、断点续传、秒传。
- 附件审批、导出、版本管理和批量处理。
- Word、Excel、PDF、图片等多类型文件合成为一个 PDF。该能力需要文件转换服务，后续单独设计。
- 知识库和 AI 对话功能。

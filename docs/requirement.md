# 需求说明

## 项目目标

建设“大同房建公寓段标准化资料管理系统”，用于统一配置科室资料目录、维护文件内容、下达公示资料，并收集车间上传的标准化资料附件。

第一阶段按真实 MVP 实施，重点保证半个月内可演示注册审批、组织管理、用户权限矩阵、首页科室文件统计、科室多级目录维护、上传任务创建、下达文件发布、车间附件填报、上传记录查看和操作日志。

## 一期范围

- 登录、注册、滑块验证码、管理员审批。
- 登录态使用 HttpOnly Cookie 保存，前端不保存登录 JWT；系统不提供跳过密码的管理员免密登录入口。
- 组织树、部门管理、用户管理、角色模板。
- 用户系统级权限配置和权限矩阵。
- 科室多级资料目录维护，最高五层，每一级可放文件夹和文件。
- 目录树下文件入口维护、文件年份由年度目录继承，并维护富文本文件内容；编辑文件时可拖拽或点击重新选择文件附件。
- 科室资料页默认按当前年份展示文件列表，并支持按年份和文件名搜索。
- 科室资料页使用统一目录展示上传任务和下达文件，不再展示“上传 / 下达”tabs。
- 上传任务文件在目录树中展示“上传”标识；文件夹默认不展示上传进度，可勾选“显示上传进度”后展示，进度只统计已配置收集项的上传任务文件。
- 文件详情支持预览 PDF、图片、Word、Excel 和 PPT；其中 PDF 和图片直接内联预览，Office 文件通过 OnlyOffice 预览服务展示。
- Office 预览使用短时附件访问票据，票据只允许读取对应附件，不暴露登录 JWT。
- 科室维护的文件正文附件同一时间只允许保留一个；如需重新上传，必须先删除原有正文附件。
- 上传任务用于科室收集车间文件；任务可指定需要提交的车间，可配置单份提交或多份提交，并配置一个或多个收集项，收集项可由任务发起者填写说明。
- 车间用户按上传任务的收集项逐项上传附件；每个收集项都上传附件后视为一次完整提交。
- 单份提交任务按车间锁定，每个选中车间最多保留一条当前有效提交；同车间普通用户可查看是否已上传但不能重复上传，科室用户、超级管理员、本车间管理员和上传本人可下载，车间侧不能删除，科室用户和超级管理员可删除用于纠错。
- 多份提交任务允许每个用户上传多条提交；上传本人可删除自己的提交，本车间管理员可删除本车间提交，科室用户和超级管理员可管理任务下全部提交。
- 科室用户和超级管理员查看本科室上传任务下全部上传记录；普通上传者只能查看和下载自己有权限的提交附件。
- 下达文件用于科室公示信息，支持富文本内容和下达附件下载，不参与上传进度统计。
- 科室文件删除后进入本科室共享回收站，回收站按文件表格展示，不做目录分级；恢复时由用户选择同科室目标目录。回收站文件保留 30 天，超过期限后系统自动清理正文附件、车间上传附件和相关元数据；单独软删除的正文附件超过 30 天后也自动清理。
- 首页展示科室组织树并统计各科室文件数。
- 首页统计、操作日志、登录日志、存储统计基础版。
- 用户侧错误提示面向普通用户，不暴露 HTTP 状态码、程序异常、端口号或底层服务名称；同类并发错误只提示一次。
- Docker Compose 部署。

## 延后范围

- 大文件分片上传、断点续传、秒传。
- 附件审批、导出、版本管理和批量处理。
- Word、Excel、PDF、图片等多类型文件合成为一个 PDF。该能力需要文件转换服务，后续单独设计。
- 知识库和 AI 对话功能。

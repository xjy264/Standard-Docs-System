# 需求说明

## 项目目标

建设“大同房建公寓段标准化资料管理系统”，用于统一配置科室资料目录、维护文件内容、下达公示资料，并收集车间上传的标准化资料附件。

第一阶段按真实 MVP 实施，重点保证半个月内可演示注册审批、组织管理、用户权限矩阵、首页科室文件统计、科室多级目录维护、上传任务创建、下达文件发布、车间附件填报、上传记录查看和操作日志。

## 一期范围

- 登录、注册、滑块验证码、管理员审批。
- 组织树、部门管理、用户管理、角色模板。
- 用户系统级权限配置和权限矩阵。
- 科室多级资料目录维护，最高五层，每一级可放文件夹和文件。
- 目录树下文件入口分为上传任务和下达文件。
- 上传任务文件在目录树中展示“上传”标识；上传 tab 下的文件夹可配置是否展示上传进度，进度只统计已配置收集项的上传任务文件。
- 上传任务用于科室收集车间文件；任务可配置单人提交或多人提交，并配置一个或多个收集项，收集项可由任务发起者填写说明。
- 车间用户按上传任务的收集项逐项上传附件；每个收集项都上传附件后视为一次完整提交。
- 单人提交任务在任意用户完整提交后锁定；多人提交任务首版同一用户只允许提交一次。
- 科室用户和超级管理员查看本科室上传任务下全部上传记录；上传者只能查看和下载自己的提交附件。
- 下达文件用于科室公示信息，支持富文本内容和下达附件下载，不参与上传进度统计。
- 首页展示科室组织树并统计各科室文件数。
- 首页统计、操作日志、登录日志、存储统计基础版。
- 用户侧错误提示面向普通用户，不暴露 HTTP 状态码、程序异常、端口号或底层服务名称；同类并发错误只提示一次。
- Docker Compose 部署。

## 延后范围

- 大文件分片上传、断点续传、秒传。
- 附件审批、导出、版本管理和批量处理。
- Word、Excel、PDF、图片等多类型文件合成为一个 PDF。该能力需要文件转换服务，后续单独设计。
- 知识库和 AI 对话功能。
